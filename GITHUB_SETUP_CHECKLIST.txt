SKY TV EPG - GITHUB SETUP CHECKLIST
===================================

1. BUILD AND SAVE APPROVED MAPPINGS
   Run the optimized Colab notebook for each server and copy these files from
   the Stage 2 output ZIPs into the repository's mappings folder:

     mappings/server_1_final_mapping.csv
     mappings/server_2_final_mapping.csv
     mappings/server_3_final_mapping.csv

2. CREATE A GITHUB REPOSITORY
   Create a repository such as skytv-epg. Upload the CONTENTS of the
   SKYTV_EPG_GitHub_Ready folder. Keep this exact workflow path:

     .github/workflows/update_epg.yml

3. ADD SERVER SECRETS
   Repository -> Settings -> Secrets and variables -> Actions.
   Add these only for servers whose mapping uses panel XMLTV:

     SERVER_1_BASE_URL      SERVER_1_USERNAME      SERVER_1_PASSWORD
     SERVER_2_BASE_URL      SERVER_2_USERNAME      SERVER_2_PASSWORD
     SERVER_3_BASE_URL      SERVER_3_USERNAME      SERVER_3_PASSWORD

   The base URL must be only http(s)://host[:port]. Do not add /xmltv.php,
   /player_api.php, username, or password to the URL.

4. ALLOW THE SUCCESS TIMESTAMP COMMIT
   Settings -> Actions -> General -> Workflow permissions ->
   Read and write permissions -> Save.

5. ENABLE PAGES
   Settings -> Pages -> Build and deployment -> Source -> GitHub Actions.

6. RUN ONCE
   Actions -> Update TiviMate EPG -> Run workflow.
   Confirm due-check, build, and deploy are green.

7. USE THE FINAL URLS IN TIVIMATE

   https://YOUR_USERNAME.github.io/YOUR_REPOSITORY/epg/server_1_tivimate.xml.gz
   https://YOUR_USERNAME.github.io/YOUR_REPOSITORY/epg/server_2_tivimate.xml.gz
   https://YOUR_USERNAME.github.io/YOUR_REPOSITORY/epg/server_3_tivimate.xml.gz

SCHEDULE
--------
The workflow checks every six hours and rebuilds only after 72 hours have
passed since the last successful Pages deployment. A manual run always builds.

SAFETY
------
The scheduled workflow never reruns fuzzy matching. It uses approved mappings
only. Never commit server credentials, playable URLs, or account tokens.
